# x402 — the payment spine

`@circuit/x402` is the heart of the SDK: a generic client that turns any `402 Payment Required` into an
on-chain CIRC micropayment and retries, plus the server-side verifier to gate your own endpoints. It
has **zero runtime dependencies** and works against *any* x402 endpoint, not just Circuit's.

## What x402 is

x402 revives HTTP's long-dormant `402 Payment Required` status as a real micropayment protocol. Instead
of an API key and a monthly invoice, **each request pays for itself, on-chain, per call.** The whole
Circuit network runs on it — inference and the data API charge identically — so a wallet is your
account *and* your meter. No sign-up, no keys, no subscription.

The settlement asset is **CIRC**, a Solana **Token-2022** mint
(`8fQgfsRnRkKSeNUhevT7wp8mhNvMSJdLn1fJi4oVpump`).

---

## The loop

| Step | What happens |
|------|--------------|
| **1 · Ask** | The client `POST`s your request with no payment. |
| **2 · 402** | The server replies `402` with a `payment` block — the **CIRC amount** and the **treasury** to send it to. |
| **3 · Pay** | The client transfers that CIRC on Solana and gets a transaction signature. |
| **4 · Retry** | It re-sends with `X-Payment-Signature: <txSig>`; the server verifies the on-chain transfer (correct mint, recipient, amount, recent, single-use) and returns the result. |

The `402` body looks like:

```json
{
  "error": "Payment Required",
  "payment": {
    "token": "8fQ…pump", "tokenSymbol": "CIRC", "tokenDecimals": 6,
    "recipient": "<treasury>", "amountRaw": "300000000", "amountDisplay": "300.00 CIRC",
    "usdEquivalent": 0.03
  }
}
```

## Paying (client)

The high-level clients (`Inference`, `Data`) build an `X402Client` for you. To pay an arbitrary
endpoint yourself:

```ts
import { X402Client } from '@circuit/x402';
import { makeWallet } from '@circuit/wallet';

const x402 = new X402Client({ wallet: makeWallet() });

// pay-and-parse JSON:
const { data, paymentTx, quote } = await x402.json('https://api.circuitllm.xyz/api/market-overview', { method: 'GET' });

// or get the raw Response:
const resp = await x402.fetch('https://api.circuitllm.xyz/api/quote');   // free → 200, no payment

// or fully generic — you control the request function:
const { resp: r } = await x402.request((extra) =>
  fetch(url, { method: 'POST', headers: { 'Content-Type': 'application/json', ...extra }, body }));
```

Free endpoints return `200` and pass straight through — `X402Client` only pays when it sees a `402`.

## Spend caps & approval

x402 is automatic, so two guardrails matter:

```ts
const x402 = new X402Client({
  wallet,
  maxSpendRaw: 500_000_000n,                       // refuse any quote above 500 CIRC (raw base units)
  onPay: (q) => console.log(`paying ${q.amountDisplay} → ${q.recipient}`),
});
```

- **`maxSpendRaw`** — a hard ceiling per call. A misquoting or hostile server gets a `SpendCapError`, and
  **the wallet is never charged.**
- **`onPay`** — an approval/notification hook (may be async). Throw inside it to abort the payment.

After paying, the client makes **one** automatic retry on a transient server error (`429`, `5xx`) — the
CIRC was already spent, so re-asking is safe.

## Streaming

`X402Client.request` is streaming-safe: on a `402` it reads only the (small) JSON challenge via
`resp.clone()`, then returns the *retry* response with its body untouched — so you can stream it. That's
exactly how `Inference.chatStream` works:

```ts
const { resp } = await x402.request((extra) =>
  fetch(url, { method: 'POST', headers: { ...headers, ...extra }, body }));
const reader = resp.body!.getReader();   // SSE stream, already paid for
```

Never use `X402Client.json()` for a streaming response — it buffers. Use `request()`.

## The wallet interface

The spine depends on **one** tiny interface, never a concrete wallet:

```ts
interface PaymentWallet {
  sendCirc(recipient: string, amountRaw: bigint): Promise<string>;   // → tx signature
  readonly address?: string | null;
}
```

`@circuit/wallet`'s `Wallet` implements it. In Python it's the `send_circ(recipient, amount_raw) -> str`
protocol. Anything matching the shape works — bring your own signer.

## Pricing helpers

```ts
import { circRawFromUsd, formatCirc, parse402, CircPriceOracle } from '@circuit/x402';

circRawFromUsd(0.03, 0.0001);      // → 300_000_000n  (whole-CIRC round-up, matches the server)
formatCirc(300_000_000n);          // → "300.00"
parse402(responseBody);            // → { recipient, amountRaw, amountDisplay, token, … } | null

const oracle = new CircPriceOracle();      // live CIRC/USD: Jupiter + 60s cache + 15-min last-known
await oracle.requiredRaw(0.03);            // raw CIRC for a $0.03 endpoint
```

`circRawFromUsd` uses the **same round-up math as the server**, so client and server always agree on the
amount.

## Gating your own endpoint (server)

Built a paid API? Verify CIRC payments with the same code Circuit uses — framework-agnostic, with the
Solana connection and replay store injected (no `@solana/web3.js` hard dependency, no Express):

```ts
import { Connection } from '@solana/web3.js';
import { verifyPaymentTx, circRawFromUsd, MemoryReplayStore } from '@circuit/x402';

const connection = new Connection(process.env.RPC_URL!);
const replay = new MemoryReplayStore();   // single-use signatures (swap in a Redis/disk store for HA)
const treasury = process.env.TREASURY!;

async function handle(req, res) {
  const requiredRaw = circRawFromUsd(0.03, await circUsd());   // your price
  const sig = req.headers['x-payment-signature'];
  if (!sig) return res.status(402).json({ payment: { token: CIRC_MINT, recipient: treasury, amountRaw: requiredRaw.toString() } });
  try {
    await verifyPaymentTx(sig, requiredRaw, { connection, treasury, replay });   // throws on any problem
    return serve(req, res);
  } catch (e) {
    return res.status(402).json({ error: 'Payment Verification Failed', reason: e.message });
  }
}
```

`verifyPaymentTx` checks: the tx is confirmed and didn't fail, it's within the age window (default 5
min), it transferred **≥ requiredRaw** CIRC to `treasury`, and the signature hasn't been used before. The
pure balance-diff (`circReceived`) is exported separately if you want to inspect a transaction yourself.

## Errors

| Error | When | Carries |
|-------|------|---------|
| `PaymentRequiredError` | a `402`, but no wallet is configured | `.quote` |
| `SpendCapError` | the quote exceeds `maxSpendRaw` (wallet **not** charged) | `.quote`, `.capRaw` |
| `X402RequestError` | the final response is non-2xx | `.status`, `.body` |

All are catchable and typed; the wallet is only ever charged once the spend cap and approval hook pass.
